# 存档功能 (Auto-Save + File Export/Import)

- **日期**: 2026-06-23
- **状态**: 已实现，待验证

## 需求

玩家未拼完的作品可以保存，下次打开继续拼。采用 **方案 A + C 组合**：
- **A**：localStorage 自动存档，启动时检测并提示恢复
- **C**：手动导出/导入 `.json` 备份文件

## 实现

### 1. 保存格式升级 (v=1 → v=2)

**`GridModel.save(appState)`** 新增 `appState` 参数，保存 v=2 格式：

```javascript
{
  v: 2, rows, cols, beads, locked, history, ts,
  app: {
    customColors,     // 自定义调色板
    artworkName,      // 作品名称
    activePaletteKey, // 当前调色板
    activeColor,      // 当前颜色
    tool,             // 当前工具
    zoomLevel,        // 缩放级别
  }
}
```

**`GridModel.load()`** 兼容 v=1 和 v=2，返回完整 data 对象供 App 恢复上层状态。

### 2. 自动存档

`App._debounceSave()` 每次触发时传入完整 app 状态。触发点：
- 放置/擦除豆子（已有）
- 清空画布（已有）
- 调整网格大小（已有）
- 熨烫完成（已有）
- **新增**：作品名称输入
- **新增**：切换调色板
- **新增**：添加/删除自定义颜色
- **新增**：缩放变更

### 3. 启动恢复流程

`App` 构造函数启动时：
1. 调用 `_getSavePreview()` 检测 localStorage 存档
2. **有存档**（含豆子）→ 显示「💾 发现未完成的创作」模态框
   - 显示作品名、网格尺寸、豆子数量、保存时间
   - 「继续创作」→ `grid.load()` + `_restoreAppState()` → 完全恢复
   - 「重新开始」→ `_startFresh()`（不删存档）
   - 「🗑 删除存档并重新开始」→ 删除 localStorage + `_startFresh()`
3. **无存档** → 直接 `_startFresh()`

### 4. 存档管理 UI

工具栏新增「💾 存档 ▼」下拉按钮（在「📷 导出」旁边）：

| 操作 | 行为 |
|------|------|
| 📥 导出存档文件 | 下载 `{作品名}_存档.json` |
| 📤 导入存档文件 | 选择 `.json` → 解析 → 恢复（有豆子时 confirm 覆盖） |
| 🗑 删除存档 | confirm → `localStorage.removeItem()` |

### 5. HTML 新增

- `#continue-modal` — 启动时存档恢复模态框
- `#btn-save-mgmt` + `#save-menu` — 工具栏存档下拉

## 代码变更

- `GridModel.save()` — 接受 `appState` 参数，升级 v=2
- `GridModel.load()` — 返回 data 对象（而非 boolean），兼容 v=1/v=2
- `App._debounceSave()` — 传入完整 app 状态
- `App._getSavePreview()` — 新增：读取存档预览
- `App._restoreAppState(data)` — 新增：从 v=2 data 恢复全部状态
- `App._startFresh()` — 新增：提取的空白启动逻辑
- `App._showContinueModal(data)` — 新增：显示继续/新建选择
- `App._buildSaveData()` — 新增：构建完整导出数据
- `App._exportSaveFile()` — 新增：下载 .json
- `App._importSaveFile()` — 新增：上传 .json 恢复
- `App._confirmDeleteSave()` — 新增：删除存档
- `_initUI()` — 新增存档下拉事件绑定 + 作品名输入监听
- `_switchPalette()` / `_addCustomColor()` / `_buildSwatches()` — 新增 `_debounceSave()` 触发
- `_initZoom()` — 新增缩放后 `_debounceSave()` 触发

## 兼容性

- 旧 v=1 存档：grid 数据正常加载，app 状态（调色板、名称等）使用默认值
- 损坏 JSON：静默忽略，正常启动空画布
- localStorage 满：`try/catch` 静默忽略，不影响使用
