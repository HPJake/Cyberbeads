# 描画功能优化

- **日期**: 2026-06-23
- **状态**: 已实现，待验证

## 需求

参考图片系统的6项优化：
1. 画布始终居中，侧控件紧密贴在画布左侧
2. 竖排透明度滑块修复（部分浏览器无法拖动）
3. 一键清除参考图（描画层+桌面框+侧控件）
4. 桌面参考框收起/展开动画（窗口飞入按钮 / 按钮飞出窗口）
5. 导入按钮换全平台 emoji + 加"参考图"文字

## 实现

### 1. 画布居中 + 侧控件定位

- `#canvas-area` 改为 `position: relative; width: 100%; display: flex; justify-content: center`
- `#ref-side-controls` 改为 `position: absolute; z-index: 10`
- 新增 `_positionSideControls()` 方法：根据 `#canvas-container` 的 `offsetLeft` 动态计算侧控件 left 位置（间距 8px）
- 在以下时机调用：`_applyReference` 导入后、窗口 resize、网格 resize、存档恢复后

### 2. 竖排滑块修复

- 移除 `writing-mode: vertical-lr; direction: rtl`
- 滑块外包裹 `<div id="ref-slider-wrap">`，用 `transform: rotate(-90deg)` 实现竖排
- 容器 `24px × 80px`，旋转后视觉竖排，浏览器正确映射鼠标事件

### 3. 清除参考图按钮

- `#ref-side-controls` 底部新增 `<button id="ref-clear-btn">✕ 清除</button>`
- `_clearReference()` 方法：
  - 清空 `_refImage`、`_refDataUrl`、`_refEnabled`、`_refCollapsed`
  - 隐藏侧控件、清空 ref-canvas、隐藏桌面框
  - 移除展开按钮（若存在）

### 4. 桌面框收起/展开动画

**交互流程**：
- 点 × → 桌面框从当前位置缩小 + 移动到侧控件中的按钮位置（300ms，ease-in-out）
- 按钮出现（CSS 画框+箭头图标）
- 点按钮 → 桌面框从按钮位置放大 + 移动到默认位置（300ms）
- 按钮消失

**新增方法**：
- `_collapseDesktopRef()` — 收起动画：逐帧移动 `left/top` + `scale(1→0)`
- `_expandDesktopRef()` — 展开动画：逐帧移动 `left/top` + `scale(0.01→1)`
- `_easeInOutCubic(t)` — 缓动函数

**CSS 图标**（`#ref-expand-btn`）：
- `::before` — 16×12px 方形边框
- `::after` — 右上角三角形箭头（border 技法）

### 5. 导入按钮

- emoji: `🖼` → `📷` (U+1F4F7, Unicode 6.0, 全平台)
- 按钮内显示 `📷 参考图`
