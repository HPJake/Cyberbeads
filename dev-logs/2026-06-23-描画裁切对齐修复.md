# 描画裁切对齐修复 + UI 微调

- **日期**: 2026-06-23
- **状态**: 已实现，待验证

## 需求

1. 裁切参考图时框住区域与实际截入区域不一致（永远偏左）
2. "描画"勾选框改名为"描图"
3. 调色板内元素（参考图按钮、预设色下拉、颜色圆框、自选色按钮）水平对齐

## 实现

### 1. 裁切偏左 Bug 修复

**根因**：`#crop-canvas-wrap` 使用 `display: flex` 居中 canvas，裁切框用绝对定位置于 wrap 中。确认裁切时直接用 `(maxCanvasW - S) / 2` 作为 canvas 位图读取坐标，未考虑 canvas 元素在 wrap 内的 CSS 布局偏移（flex 居中、overflow 裁剪等造成的偏移）。

**修复**：裁切确认时改用 `getBoundingClientRect()` 精确计算裁切框相对于 canvas 位图的真实位置：

```js
const canvasRect = canvas.getBoundingClientRect();
const overlayRect = overlay.getBoundingClientRect();
const sx = overlayRect.left - canvasRect.left;
const sy = overlayRect.top - canvasRect.top;
const scaleX = canvas.width / canvasRect.width;
const scaleY = canvas.height / canvasRect.height;
cctx.drawImage(canvas,
  sx * scaleX, sy * scaleY,
  S * scaleX, S * scaleY,
  0, 0, S, S);
```

同时处理了 devicePixelRatio 缩放（canvas 位图尺寸 vs CSS 显示尺寸）的边界情况。

### 2. "描画" → "描图"

`#ref-toggle` 勾选框标签文案从 `描画` 改为 `描图`。

### 3. 调色板水平对齐

| 位置 | 变更 |
|------|------|
| 外层 wrapper `<div>` | `align-items: flex-start` → `align-items: center` |
| `#palette` CSS | 新增 `align-items: center` |
| `#btn-custom-color` | 移除 `margin-top:4px` |

确保参考图按钮、预设色下拉框、颜色圆框、自选色按钮的中心点在同一水平线上。

### 4. 预设色调色板下拉按钮偶发无响应

**根因**：`_buildPaletteDropdown()` 在初始化（`_init`）和恢复存档（`_restoreAppState`）时各调用一次，每次都给 `#palette-current` 添加一个 `pointerdown` 监听器。两个监听器都执行 `classList.toggle('open')`，toggle 两次 = 打开后立即关闭 = 看起来无响应。

这解释了为什么 Edge "有时候"不响应——取决于浏览器是否有存档数据触发了 `_restoreAppState` 流程。Chrome 若无存档则只绑定一次，正常。

**修复**：加 `_paletteDropdownBound` 标记，确保 `#palette-current` 的 toggle 监听器和 `document` 的 outside-click 关闭监听器只绑定一次。

同时将 document 监听器改为 `contains()` 检测（检查点击是否在 `#palette-selector` 内），替代依赖 `stopPropagation()` 的方式，更健壮、跨浏览器兼容性更好。
