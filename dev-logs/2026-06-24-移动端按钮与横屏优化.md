# 2026-06-24 移动端按钮与横屏优化

## 需求

1. 手机横屏时也应用竖屏的紧凑布局（而非更紧凑的横屏专属规则）
2. 手机端"存档"只保留图标、"参考图"保留文字、"布置桌面"显示图标+文字"布置"
3. 全屏按钮移到静音按钮上方，同一列置于页面右上角

## 改动

### `index.html`

- `#btn-save-mgmt`：添加 `btn-icon-only` 类 → 手机端只显示 💾 图标
- `#btn-import-ref`：移除 `btn-icon-only` 类 → 手机端保留 "参考图" 文字
- `#btn-desk-arrange`：文字从 "布置桌面" 改为 "布置"

### `css/style.css`

- **媒体查询扩展**：`@media (max-width: 480px)` → `@media (max-width: 480px), (orientation: landscape) and (max-height: 500px)`
  - 横屏手机（宽度 > 480px 但高度 ≤ 500px）也会应用紧凑布局
- **删除横屏专属规则**：移除 `@media (max-width: 480px) and (orientation: landscape)` 整块（之前把按钮缩到 24px 的更紧规则）
- **全屏按钮**：`right: 40px` → `right: 6px`（移到最右侧，与静音同列）
- **静音按钮**：`top: 6px` → `top: 38px`（移到全屏按钮下方）
- **布置桌面按钮**：
  - 删除 `width: 32px; height: 32px; border-radius: 50%` 圆形规则
  - 删除 `.btn-text { display: none; }` 文字隐藏
  - 改为胶囊形：`height: 28px; padding: 0 10px; border-radius: 14px; font-size: 12px`

## 右上角按钮布局（手机端）

```
┌─────┐
│ ⛶  │  全屏 (top: 6px, right: 6px)
└─────┘
┌─────┐
│ 🔊  │  静音 (top: 38px, right: 6px)
└─────┘
```

## 桌面端影响

零。所有改动在手机媒体查询内。

## 2026-06-24 补充修复

### 改动

1. **全屏按钮移到 `<body>` 层级**
   - 原来嵌套在 `#toolbar` 内，`position: fixed` 可能受父元素影响
   - 现移到 `<body>` 下，与 `#mute-btn` 同级 → 固定定位万无一失

2. **调色栏修复**
   - `flex-wrap: nowrap; overflow-x: auto` → `flex-wrap: wrap`
   - 手机窄屏下允许换行，不再挤压成细条

3. **参考图按钮手机端去图标**
   - ⛰️ 图标用 `<span class="btn-icon">` 包裹
   - 手机端 `#btn-import-ref .btn-icon { display: none; }` → 只显示"参考图"文字
   - 桌面端不变：⛰️ 参考图

4. **布置桌面按钮文字切换改用 `::after` 伪元素**
   - 之前用双 span 切换，出现两个文字同时显示的 bug
   - 现用 `::after { content: '...' }` — 桌面 `布置桌面`，手机 `布置`
   - 单元素，无冲突可能
