# 2026-06-25 抓手工具（Pan Tool）

## 需求

画布放大后，点击画布内部会被镊子/豆笔拦截，无法拖动页面。新增「抓手」工具，选中后可在画布内长按拖动，像滚轮一样滚动整个页面。

## 实现方式

**双层结构**：外层 `#canvas-zoom-wrapper` 提供真实布局尺寸 → `#app` 产生滚动条；内层 `#canvas-container` 使用 `transform: scale()` 保持渲染清晰。

- 外层 CSS 尺寸 = 内部尺寸 × zoom → 真正撑大布局
- 内层 `transform: scale(zoom); transform-origin: 0 0;` → 豆子/格子渲染保持清晰
- 抓手工具滚动 `#app.scrollLeft` / `#app.scrollTop`

## 改动

### `index.html`
- `#canvas-container` 外层包裹 `#canvas-zoom-wrapper`
- `#tool-drawer` 新增 `#tool-hand` 工具卡片

### `css/style.css`
- 视觉样式（border-radius、box-shadow、cursor）移到 `#canvas-zoom-wrapper`
- `#canvas-container` 仅保留 `position: relative; touch-action: none; transform-origin: 0 0;`
- canvas 元素不设 `width`/`height`（使用 intrinsic size）
- `#app` 改为 `overflow: auto`（支持双向滚动）
- `.hand-cursor` / `.grabbing` 光标类应用于 wrapper

### `js/app.js`
- 新增 `_applyZoom()`：设置 wrapper CSS 尺寸 + container 的 `transform: scale(zoom)` 和 CSS 宽高
- `_initZoom`：滚轮/捏合事件监听移至 wrapper
- 每次 `renderer.resize()` 后调用 `_applyZoom()`
- `_positionSideControls`：基于 wrapper 的 offsetWidth 定位

### `js/tool-drawer.js`
- 手掌图标、三选一互斥、cursor class 作用于 wrapper

### `js/input-manager.js`
- 事件监听移至 wrapper（覆盖完整缩放区域）
- 抓手拖动操作 `#app.scrollLeft/Top`
- 优先级：ironing > hand > tweezers/beadpen

### `js/renderer.js`
- `drawToolCursor`：hand 工具跳过自定义光标

## 交互

- 缩放：滚轮/捏合 → wrapper 变大 → `#app` 自然产生滚动条
- 抓手：在画布任意位置按住拖动 → `#app` 滚动，整个页面移动
- 渲染：`transform: scale()` 保持豆子清晰，不拉伸模糊
