# 屏幕取色功能

- **日期**: 2026-06-23
- **状态**: 已实现，待验证

## 需求

在"自选色"展开窗口中加入屏幕取色按钮，读取屏幕任意像素颜色，自动匹配最近 MARD 291 色号，取色后颜色选择器同步更新。

## 实现

### HTML

在 `#color-picker-panel` 的 MARD 色号输入框下方、应用按钮上方添加：

```html
<button class="btn" id="picker-eyedropper-btn" title="从屏幕取色"
        style="width:150px;height:28px;font-size:12px;">💧 屏幕取色</button>
```

### JS

使用浏览器原生 [EyeDropper API](https://developer.mozilla.org/en-US/docs/Web/API/EyeDropper)（Chrome 95+ / Edge 95+）：

1. **特性检测**：检查 `typeof EyeDropper`，不支持则禁用按钮并更新 title 提示
2. **取色流程**：
   - `new EyeDropper().open()` → 用户在屏幕上点选颜色
   - 返回 `{ sRGBHex }` → 调用 `findClosestMardCode(pickedHex)` 匹配 MARD 291 色号
   - 更新 `#mard-code-input` 显示匹配色号
   - 调用 `_pickerFromHex(hex)` + `_updatePickerUI(...)` 同步颜色选择器
3. **取消处理**：用户按 ESC 取消时抛出 `AbortError`，静默忽略

## 用户体验

- 点击"💧 屏幕取色"→ 光标变为取色器 → 点屏幕任意位置
- 取色后自动匹配 MARD 291 色表中最接近的颜色
- 色号输入框更新为匹配的 MARD 色号
- 颜色选择器（饱和度/明度/色相）同步更新
- 用户确认无误后点"应用"添加到自定义调色板
