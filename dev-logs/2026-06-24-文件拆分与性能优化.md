## 二、性能优化

### 优化 1：熨烫温度扫描 O(n²) → O(1)

**问题**：`IroningSystem.update()` 每帧遍历全部格子查找最高热度，100×100 网格 = 每帧 10,000 次访问。

**修复**：增量追踪最高热度
- `constructor()` 新增 `this.maxHeat = 0`
- `heatCell()` 中每次加热后更新：`this.maxHeat = Math.max(this.maxHeat, bead.heat)`
- `update()` 直接使用 `this.maxHeat`，删除全网格扫描
- `start()` / `cancel()` 时重置 `this.maxHeat = 0`

**原理**：熨烫过程中热量只增不减（加热到 1.0 即铁定），增量追踪完全准确。

**影响文件**：`js/ironing-system.js`（~6 行新增/修改）

### 优化 2：指针事件节流

**问题**：非熨烫模式下 `pointermove` 每 ~1ms 触发一次 `_onMove()`，大量无效的 `_getCell()` 和同格检查。

**修复**：`_onMove()` 添加 12ms 时间阈值
- `constructor()` 新增 `this.lastMoveTime = 0`
- 非熨烫模式：距上次处理 < 12ms → 跳过（~83Hz，仍高于 60fps）
- 熨烫模式：保持全频率（需要插值精度）

**影响文件**：`js/input-manager.js`（~4 行新增/修改）
